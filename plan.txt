2. LIVE WEBSOCKET SUBSCRIBER ENGINE
--------------------------------------------------------------------------------
The application will establish a persistent connection to Deriv's official production/test 
WebSocket server to stream ticks in real time without any mock data or database simulations.

- Correct Endpoint: wss://ws.derivws.com/websockets/v3?app_id=1089 (Confirmed official connection url)
- Subscription requests:
  - Subscription payload:
    {
      "ticks": "R_100",
      "subscribe": 1
    }
- Ping Protocol & Latency Tracking:
  - Periodic heartbeat messages are sent (every 10 seconds):
    {
      "ping": 1
    }
  - Latency is calculated between the send timestamp and execution timestamp of the matching response.
  - Connection indicator color-coding system:
    - BLUE (-best-): Latency < 100 ms
    - GREEN (-fair-): Latency 100 ms to 250 ms
    - ORANGE (-poor-): Latency 250 ms to 500 ms
    - RED (-unstable-): Latency > 500 ms or disconnected

3. ALGORITHMIC ANALYSIS ENGINE (NO SIMULATIONS)
--------------------------------------------------------------------------------
The analytics engine listens to the live tick stream and applies four exact rules:

A. Over/Under Trend Analysis
- Base scope: Calculates conditions using precisely the last 100 ticks.
- Determines overall trend direction, average deviation, and frequency of ticks exceeding or falling below the specified barrier (0-9 range).

B. Two-Profile Comparison Matches
- Operates on dual statistical windows concurrently:
  - Profile A: Last 50 ticks
  - Profile B: Last 25 ticks
- Evaluates similarities and divergence between frequencies of the digits (0-9).
- Produces match predictions associated with a dynamic confidence score calculated from consistency between the two windows.

C. Digit Balancing Rule
- Based on the phenomenon that over large sample sizes, digits 0-9 strive for a 10% frequency balance.
- If a digit has been heavily underrepresented/absent for consecutive cycles, its predicted probability of appearing increases to represent the expected correction.
- Conversely, highly overrepresented digits trigger a downward dampening adjustment on the predictor.

D. Repeating Sequence Pattern Predictor
- Uses a local rolling sequence database storing exactly the last 1000 tick patterns for the currently selected symbol.
- Every new live tick appends to this persistent local storage, and older ticks are automatically pruned to guarantee that the database size is kept strictly at 1000 ticks.
- When a 4-digit sequence (e.g., "1614") occurs from the latest live ticks, the pattern engine queries the 1000-tick historical database to scan for previous occurrences of this prefix context (e.g., identifying when "1614" emerged in the past and was followed by "1", then "3", then "7").
- If a match is found within this historical sliding window, the engine projects the future succession sequence predictions ("1", then "3", then "7") for visual exact match entries, assigning higher priority/confidence score to these outcomes in the "MATCHES" signals list.

4. CORE SYSTEM PERMISSIONS & CONVENTIONS
--------------------------------------------------------------------------------
The application requests the following structural Android permissions:
- android.permission.INTERNET (Essential for WebSocket streaming)
- android.permission.ACCESS_NETWORK_STATE (To monitor connection state changes)
- android.permission.REQUEST_IGNORE_BATTERY_OPTIMIZATIONS (Ensures WebSocket is not closed during background/idle)
- android.permission.SYSTEM_ALERT_WINDOW (Allows drawing over other apps if custom PiP overlays are triggered)

--------------------------------------------------------------------------------
5. PICTURE-IN-PICTURE (PiP) ARCHITECTURE
--------------------------------------------------------------------------------
To monitor live analytics even when exiting ProTrader, the primary screen supports standard Android Picture-in-Picture.
- Manifest setup:
  <activity
      android:name=".MainActivity"
      android:supportsPictureInPicture="true"
      android:configChanges="screenSize|smallestScreenSize|screenLayout|orientation">
- Activation: Emits `enterPictureInPictureMode(PictureInPictureParams.Builder().build())` inside `onUserLeaveHint()`.
- Layout adaptation:
  - Automatically hides heavy visual boards (Digit Frequency chart, current streaks tables, parameters fields) during PiP mode using Compose context properties layout checking (via local configuration or size changes).
  - Shrinks down to focus on:
    - Current Active Symbol & Live Tick Quote (e.g., "R_100 LIVE 359.85")
    - Last Digit Highlight
    - TOP Predictions (Digit paths for MATCHES or DIFFERS)
    - Ping Latency visual dot with precise color indicator (Blue, Green, Orange, Red)
